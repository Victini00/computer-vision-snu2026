# Computer Vision — 한 학기 정리 노트

> 한 학기 동안 배운 컴퓨터 비전 전체를 "2D 이해 → 3D 복원"의 흐름으로 정리한 노트.
> 선형대수를 따로 안 들었다는 전제로, **직관 → 수학 원리 → 강의 간 연결**의 순서로 적었다.

---

## 0. 전체 지도 (한 장으로 보는 한 학기)

이 과목은 결국 하나의 질문을 점점 깊게 파고든다:

> **"2D 픽셀 배열일 뿐인 이미지에서, 우리는 무엇을 알아낼 수 있는가?"**

- **중간고사 범위 (2D 세계):** 이미지 한 장 안에서 *무엇이 어디 있는가*를 본다.
  필터링 → 엣지 → 코너/특징점 → 매칭 → 인식 → 추적 → 분할.
- **기말고사 범위 (3D 세계):** 여러 장의 이미지로 *세상이 실제로 어떤 3D 구조인가*를 복원한다.
  투영기하 → 카메라 모델 → 캘리브레이션 → 자세추정 → 다시점 기하 → SfM/BA → 스테레오.

**핵심: 2D는 "이미지를 설명"하고, 3D는 "이미지를 만든 세상을 거꾸로 푼다".**

한 학기 전체를 관통하는 5가지 도구(아래 [부록 A](#부록-a-한-학기를-관통하는-5가지-핵심-패턴))를 먼저 머리에 넣어두면, 모든 강의가 그 변주처럼 보인다.

---

## Part I. 수학 기초 (Lectures 2–3)

선형대수를 안 들었어도 이 과목은 사실 **딱 몇 개의 도구**만 반복한다.

### 행렬 = "변환기"
행렬 `A`를 벡터 `x`에 곱하는 것(`Ax`)은 **x를 다른 곳으로 옮기는 기계**라고 보면 된다. 회전, 확대, 찌그러뜨리기 전부 행렬 곱으로 표현된다.

### 고유값/고유벡터 (eigenvalue/eigenvector)
- **직관:** 행렬이라는 기계에 넣어도 *방향이 안 바뀌고 길이만 변하는* 특별한 화살표가 고유벡터. 그 길이 변화 배율이 고유값.
- **왜 중요한가:** "데이터가 주로 어느 방향으로 퍼져 있나?"를 알려준다. → 나중에 PCA, 코너 검출, optical flow에서 똑같이 등장한다.

### SVD (특이값 분해)
- **직관:** 어떤 행렬이든 `A = UΣVᵀ`로 쪼갤 수 있다. "회전 → 축별 크기조절 → 회전" 3단계로 분해한 것.
- **이 과목에서의 진짜 용도:** `Ax = 0`을 푸는 만능 레시피. → [부록 A](#부록-a-한-학기를-관통하는-5가지-핵심-패턴) 참고.

**기억할 것: 이 과목의 수학 90%는 "Ax=0 풀기"와 "고유벡터 찾기"로 환원된다.**

# 중간고사 범위 — 2D 이미지의 세계

## Part II. 이미지 형성과 필터링 (Lectures 4–5)

### 필터링 = 이웃을 보고 픽셀을 다시 쓰기
- **상관(correlation):** 필터(커널)를 이미지 위에 올려놓고, 겹치는 픽셀끼리 곱해서 더한다. 본질적으로 **내적(dot product) = 두 패턴이 얼마나 닮았는지 측정**.
- **합성곱(convolution):** 필터를 상하좌우로 뒤집은 뒤 상관 연산. 필터가 대칭이면 둘은 동일.
- **용도별 필터:**
  - Box / Gaussian → **블러(denoising)**. Gaussian은 가운데에 가중치를 더 주는 "가중 평균".
  - Sharpening → 중심과 이웃의 차이를 강조.
  - 미분 필터 `[-1, 0, 1]` → **밝기 변화율(=엣지)** 측정.

### 이미지 그래디언트 (∇I)
- **직관:** 각 픽셀에서 "어느 방향으로 가면 밝기가 가장 급하게 바뀌나"를 가리키는 화살표.
- **크기 `‖∇I‖`** = 엣지의 강도, **방향** = 엣지에 수직인 방향.
  - `‖∇I‖ = √((∂I/∂x)² + (∂I/∂y)²)`

### Canny Edge Detector (엣지 검출의 고전)
4단계 파이프라인:
1. **Gaussian 블러** (노이즈 제거, σ가 클수록 굵은 엣지만 남음)
2. **그래디언트** 크기·방향 계산
3. **Non-Maximum Suppression (NMS):** 그래디언트 방향으로 봤을 때 가장 센 픽셀만 남겨 엣지를 1픽셀로 얇게.
4. **Hysteresis (이중 임계값):** 높은 임계값을 넘는 "강한 엣지"에서 출발해, 거기에 연결된 "약한 엣지"까지 살린다.

**기억할 것: 필터링은 "이웃과의 내적", 엣지는 "밝기의 1차 미분".**

## Part III. 특징점: 코너와 SIFT/HOG (Lectures 6–8)

엣지만으로는 매칭이 어렵다(엣지는 선 따라 미끄러진다 = aperture problem). 그래서 **딱 짚어 찾을 수 있는 점(=코너)** 과 **그 점을 설명하는 지문(=descriptor)** 이 필요하다.

### 코너 검출 (Harris)
- **직관:** 작은 창을 살짝 움직였을 때
  - 평평한 곳 → 변화 없음
  - 엣지 → 한 방향으로만 변함
  - **코너 → 모든 방향으로 변함** ← 이게 좋은 특징점.
- **수학:** 창 안의 그래디언트들로 만든 2×2 구조 행렬(structure tensor)의 **두 고유값이 모두 크면 코너**.
  → Part I의 고유값이 여기서 처음 실전 등장. (나중에 optical flow에서 똑같은 행렬이 또 나온다!)

### 좋은 특징(feature)의 조건
조명·색·**스케일·회전**이 바뀌어도 똑같이 잡히고(repeatable) 똑같이 설명되는(invariant) 것.

### SIFT (Scale-Invariant Feature Transform) [Lowe]
스케일·회전 불변 특징의 결정판. 4단계:
1. **검출 (위치):** Difference of Gaussian(DoG)으로 여러 스케일에서 극값을 찾음.
2. **스케일 결정:** 가장 "반복 가능한" 스케일 선택 → 멀리서 찍든 가까이서 찍든 같은 점.
3. **방향 (orientation):** 주변 그래디언트의 대표 방향을 기준으로 정렬 → 회전 불변.
4. **기술자 (descriptor):** 주변을 셀로 나눠 각 셀의 그래디언트 방향 히스토그램을 모아 128차원 벡터.

### HOG (Histogram of Oriented Gradients)
- 이미지를 셀로 나누고 각 셀의 **그래디언트 방향 히스토그램**을 만든다.
- SIFT가 "한 점의 지문"이라면 HOG는 "영역 전체의 모양 요약" → 보행자 검출 등에 사용.

**기억할 것: 코너 = 두 고유값이 모두 큰 점. SIFT/HOG = 그래디언트 방향 히스토그램으로 만든 지문.**

## Part IV. 매칭과 인식 (Lectures 9–12)

### Hough Transform — 점들을 모아 선/도형 찾기 (Lecture 9)
- **직관 (투표!):** 엣지 점 하나하나가 "내가 속할 수 있는 직선 후보들"에 한 표씩 던진다. 표가 가장 많이 쌓인 곳이 실제 직선.
- 이미지 공간의 점 ↔ 파라미터 공간 `(m, c)` 또는 `(ρ, θ)`의 곡선. 누적 배열(accumulator)에서 지역 최댓값을 찾으면 직선이 검출된다.
- **연결:** Canny로 엣지 뽑고 → Hough로 직선 fitting. (Part II의 직접적 후속)

### Bag of Words (BoW) — 이미지 인식 (Lecture 10)
- **직관:** 문서를 "단어 빈도"로 요약하듯, 이미지를 "시각 단어(visual word) 빈도"로 요약.
- 절차: SIFT/HOG 특징들을 모아 → 군집화(클러스터)로 **시각 사전(visual dictionary)** 생성 → 이미지를 단어 히스토그램으로 표현 → 분류.

### Face Detection (Lecture 11)
- 슬라이딩 윈도우 + 빠른 특징(Haar-like) + 캐스케이드 분류기로 "얼굴/비얼굴"을 실시간 판별 (Viola-Jones 계열).

### PCA & Eigenface (Lecture 12)
- **직관:** 데이터가 가장 많이 퍼진 방향(축)을 찾아 그 축들로만 데이터를 표현 → **차원 축소**.
- **수학:** 0-중심화한 데이터 행렬을 SVD(`A = UΣVᵀ`) → `U`의 앞쪽 열들이 주축. 각 데이터는 *소수의 주축들의 선형 결합*으로 근사.
  - 첫 번째 축 = 분산 최대 방향(가장 큰 고유벡터), 두 번째 축 = 그에 직교하면서 남은 분산 최대 … (실대칭 행렬의 고유벡터는 서로 직교).
- **Eigenface:** 얼굴 이미지들을 PCA → 소수의 "고유 얼굴"의 조합으로 모든 얼굴을 표현/인식.
- **연결:** Part I의 고유값/SVD가 인식 문제에서 정점을 찍는 지점.

**기억할 것: Hough=투표, BoW=시각 단어 빈도, PCA=가장 많이 퍼진 방향만 남기기(SVD).**

## Part V. 추적과 분할 (Lectures 13–14)

### Tracking — Lucas-Kanade Optical Flow (Lecture 13)
- **목표:** 다음 프레임에서 각 픽셀이 어디로 움직였나 `(u, v)`.
- **밝기 항상성 가정:** 움직여도 픽셀 밝기는 그대로 → `Ix·u + Iy·v + It = 0`.
- 방정식 1개에 미지수 2개 → **작은 창 안에서 같은 흐름이라 가정**해 여러 식을 모으고 최소제곱으로 푼다:
  - `(AᵀA) [u v]ᵀ = Aᵀb`, 여기서 `AᵀA`는 **Harris의 구조 행렬과 같은 2×2 행렬!**
  - 두 고유값이 모두 커야(=코너 영역) 흐름이 안정적으로 풀림. 엣지/평탄 영역은 ill-conditioned.
- **연결:** Part III의 코너 = optical flow가 잘 풀리는 점. 같은 수학이 추적에서 재등장.

### Segmentation (Lecture 14)
- 이미지를 의미 있는 영역으로 나누기. 색/질감 유사도, 그래프 기반 분할 등.

**기억할 것: optical flow는 "밝기 보존 + 작은 창 최소제곱"이고, 그 핵심 행렬은 Harris와 동일하다.**

# 기말고사 범위 — 2D에서 3D로

> 여기서부터 패러다임이 바뀐다. 이전까지는 "이미지를 설명"했다면,
> 이제는 **"이미지를 만들어낸 3D 세상과 카메라를 거꾸로 푼다".**

## Part VI. 투영 기하학 (Lectures 15–18)

3D를 다루려면 먼저 **무한대와 원근**을 수학적으로 다룰 언어가 필요하다.

### 동차좌표 (Homogeneous Coordinates)
- **문제:** 유클리드 기하는 "무한대(평행선이 만나는 곳)"를 표현 못 한다.
- **해법:** 차원을 하나 더 붙인다. 2D 점 `(x, y)` → `(x, y, 1)`. 일반적으로 `(x, y, w)`는 `(x/w, y/w)`를 의미.
- **up-to-scale:** `(x, y, w)`와 `(kx, ky, kw)`는 **같은 점**. (스케일은 무시) → 이게 한 학기 내내 따라다니는 규칙.
- **무한점:** `w = 0`이면 무한히 먼 점(방향). 평행선들이 만나는 소실점(vanishing point)이 바로 이것.

### 변환의 위계 (Hierarchy of Transformations)
점점 더 자유로워지는 순서:
1. **Euclidean** (회전+이동): 거리·각도 보존
2. **Similarity** (+크기): 각도·비율 보존
3. **Affine** (+찌그러짐): 평행선 보존
4. **Projective / Homography** (가장 일반): 직선만 보존 (평행선은 안 보존 → 원근 효과!)

### Homography (호모그래피) — 3×3 행렬 H
- **직관:** 한 평면을 다른 시점에서 본 것끼리의 매핑. 바닥 타일을 정면 vs 비스듬히 본 변환.
- `x' = Hx` (동차좌표, up-to-scale).
- **추정 (DLT):** 대응점 쌍들로부터 H를 푼다. `x' × Hx = 0` (외적으로 스케일 모호성 제거) → `Ah = 0` 형태 → **SVD로 풀기** ([부록 A](#부록-a-한-학기를-관통하는-5가지-핵심-패턴)).
- **주의:** 렌즈 왜곡(barrel/pincushion)은 호모그래피로 표현 안 됨(직선이 휜다).

**기억할 것: 동차좌표 = 무한대를 다루려 차원 하나 추가 + up-to-scale. Homography = 평면↔평면 3×3 매핑, DLT로 추정.**

## Part VII. 카메라 모델 (Lectures 19–20)

이제 3D 점이 어떻게 2D 픽셀이 되는지를 행렬 하나로 쓴다.

### 핀홀 카메라와 투영 행렬 P
3D 점 `X` → 2D 픽셀 `x`로 가는 과정을 분해:

```
x = P X,   P = K [R | t]   (3×4)
```

- **외부 파라미터 `[R | t]` (extrinsic):** 세상 좌표를 *카메라가 보는 좌표*로 옮기는 회전 `R`과 이동 `t`. → "카메라가 어디서 어느 쪽을 보고 있나".
- **내부 파라미터 `K` (intrinsic):** 카메라 내부 성질. 초점거리(focal length), 주점(principal point), 픽셀 스케일. → "카메라 렌즈/센서의 고유 스펙". 보통 상삼각(upper-triangular) 행렬.

### 카메라 중심과 역투영
- **카메라 중심 C:** `P`의 null space (즉 `PC = 0`인 점). 모든 광선이 모이는 곳.
- **역투영 (back-projection):** 픽셀 하나는 3D에서 *한 점*이 아니라 *한 광선*에 대응 (깊이를 모르니까). 의사역행렬(pseudo-inverse) `P⁺`로 광선을 복원.
- **RQ 분해:** 주어진 `P`에서 `K`(상삼각)와 `R`(직교)를 다시 분리해내는 도구. (SVD의 사촌)

**기억할 것: P = K[R|t]. K=카메라 내부 스펙, [R|t]=카메라의 위치·방향. 픽셀 하나 ↔ 3D 광선 하나.**

## Part VIII. 카메라 캘리브레이션 (Lecture 21)

`P`(특히 `K`)와 렌즈 왜곡을 **실제로 측정해 알아내는** 과정.

### DLT 캘리브레이션
- 3D 좌표를 아는 점들(예: 체커보드 코너)과 그 이미지 픽셀의 대응으로 `P`를 추정.
- 또 `x × PX = 0` → `Ap = 0` → **SVD** (또 등장!).

### Zhang's Method (체커보드 방법)
- 평면 체커보드를 여러 각도로 찍어 각 사진마다 **호모그래피 H**를 구하고 (Part VI), 그 H들로부터 `K`를 분리.
- 실무에서 가장 널리 쓰는 캘리브레이션.

### 렌즈 왜곡 (Lens Distortion)
선형 모델로 못 잡는 휘어짐을 별도 보정:
- **방사 왜곡 (radial)** `k₁, k₂, k₃`: 중심에서 멀수록 휘는 정도(barrel/pincushion).
- **접선 왜곡 (tangential)** `p₁, p₂`: 렌즈와 센서가 완전히 평행하지 않아 생기는 비대칭 왜곡.

**기억할 것: 캘리브레이션 = 알고 있는 3D-2D 대응으로 P와 왜곡을 역산. Zhang=체커보드+호모그래피.**

---

## Part IX. 자세 추정 — PnP (Lecture 22)

**상황:** `K`는 이미 안다(캘리브레이션 됨). 3D 점들과 그 이미지 픽셀만 보고 **카메라가 지금 어디서 보고 있나(`R`, `t`)** 를 구한다.

- **P3P:** 최소 3개의 3D-2D 대응으로 자세를 푸는 기하 해법.
- **EPnP:** n개의 점을 효율적으로 처리하는 방법(소수의 control point의 가중합으로 표현).
- **RANSAC 파이프라인:** 대응 중 잘못된 매칭(outlier)이 섞여 있으므로,
  1. 무작위로 소수 샘플 뽑아 자세 추정
  2. 그 자세로 나머지가 얼마나 잘 맞나(inlier 수) 평가
  3. 여러 번 반복 후 inlier 최대인 해 채택 → 이후 inlier 전체로 정밀화.
- **연결:** RANSAC은 캘리브레이션·PnP·다시점 기하 어디서나 outlier 처리에 재등장 ([부록 A](#부록-a-한-학기를-관통하는-5가지-핵심-패턴)).

**기억할 것: PnP = K를 알 때 3D-2D 대응으로 카메라 자세 [R|t]를 구하기. outlier는 RANSAC으로.**

---

## Part X. 다시점 기하와 삼각측량 (Lecture 23)

이미지 **두 장**의 관계로 들어간다.

### 에피폴라 기하 (Epipolar Geometry)
- **직관:** 왼쪽 이미지의 한 점에 대응하는 오른쪽 점은 아무 데나 있는 게 아니라 **특정 직선(epipolar line) 위**에만 있다. → 매칭 탐색이 2D에서 1D로 줄어든다.

### Fundamental Matrix F
- 두 이미지의 대응점을 잇는 제약: `x'ᵀ F x = 0`.
- **유도 직관:** 왼쪽 픽셀 `x`를 역투영(`P⁺x`, Part VII)해 광선을 만들고, 그 광선 위 점을 오른쪽 카메라로 투영하면 오른쪽의 epipolar line이 나온다. (※ `P'P⁺x`가 오른쪽에서의 대응점 역할을 한다 — 역투영된 광선상의 한 점을 오른쪽 이미지에 다시 투영한 것)
- 추정: 대응점들로 `Af = 0` → **SVD** (또!).

### 삼각측량 (Triangulation)
- 두 카메라에서 같은 점을 본 두 광선의 교차점으로 **3D 좌표 복원**.
- 2단계 패턴:
  1. **DLT**로 선형 초기해 (`x × PX = 0` → `AX = 0` → SVD)
  2. **재투영 오차(reprojection error) 최소화**로 비선형 정밀화.

**기억할 것: F는 "대응점은 epipolar line 위에 있다"는 제약. 삼각측량 = 두 광선의 교차로 3D 복원. (DLT 초기화 → 비선형 정밀화 패턴 또 등장)**

## Part XI. SfM & Bundle Adjustment (Lecture 24)

두 장을 넘어 **수십~수백 장**으로 확장.

### Structure-from-Motion (SfM)
- 여러 이미지에서 특징 매칭 → 두 장씩 자세·구조 복원 → 점진적으로 카메라들과 3D 점들을 모두 누적해 전체 장면을 복원.
- 출력: 모든 카메라 자세 `{Pᵢ}` + 3D 점 구름 `{Xⱼ}`.

### Bundle Adjustment (BA)
- **직관:** 누적되며 쌓인 오차를, 모든 카메라와 모든 3D 점을 **동시에** 미세조정해 줄이는 거대한 최적화.
- 목적함수: 전체 **재투영 오차의 합** 최소화 (Part X의 정밀화를 전 장면 규모로).
- 비선형 최소제곱(Levenberg-Marquardt 등)으로 푼다.

**기억할 것: SfM=많은 사진으로 카메라+3D점 동시 복원, BA=전체 재투영 오차를 한꺼번에 최소화하는 마무리 최적화.**

---

## Part XII. 스테레오 매칭 (Lecture 25)

**상황:** 두 카메라의 상대 위치를 안다(rectified). 모든 픽셀의 깊이(dense depth)를 구한다.

### 대응 탐색과 disparity
- 에피폴라 기하 덕에 대응점은 같은 수평선(scanline) 위에 있다 → 좌우 픽셀의 가로 위치 차이 `disparity`가 깊이에 반비례. (가까울수록 많이 어긋남)

### 에너지 최소화 (Energy Minimization)
- 좋은 disparity 맵 = **데이터 항**(좌우 픽셀이 닮아야) + **평활 항**(이웃끼리 깊이가 비슷해야)의 균형.
- 이 에너지를 최소화해 전역적으로 매끄러운 깊이 맵을 얻는다.

### Binary Structured Light
- 매칭이 어려운 무늬 없는 표면에, **알려진 패턴(줄무늬 등)을 투사**해 대응을 강제로 만들어 깊이를 측정 (능동형 방식).

**기억할 것: 스테레오 = disparity로 깊이 복원. 에너지 최소화(데이터+평활)로 dense map, 무늬 없으면 structured light로 패턴을 만들어 준다.**

# 부록 A. 한 학기를 관통하는 5가지 핵심 패턴

이 5개만 손에 익으면 모든 강의가 같은 노래의 변주로 들린다.

### 1. `Ax = 0` → SVD의 V 마지막 열
제약된 최소제곱의 만능 레시피. `‖x‖=1` 제약 아래 `‖Ax‖`를 최소화하는 `x`는 `A = UΣVᵀ`에서 **가장 작은 특이값에 대응하는 V의 마지막 열**.
> 등장: Homography(DLT), 캘리브레이션, Fundamental Matrix, 삼각측량 … 거의 전부.

### 2. 외적 트릭 (cross-product) `x × PX = 0`
동차좌표는 up-to-scale라 `x`와 `PX`는 "스케일 빼고" 같아야 한다. 같은 방향 벡터의 외적은 0 → 스케일 모호성을 깔끔히 제거해 선형 방정식으로 만든다.

### 3. DLT 초기화 → 재투영 오차 비선형 정밀화 (2단계 파이프라인)
먼저 선형(DLT)으로 빠른 근사해를 얻고, 그걸 출발점으로 비선형 최적화(재투영 오차 최소화)로 정밀화.
> 등장: 삼각측량, PnP, BA, 캘리브레이션.

### 4. 동차좌표 & up-to-scale 규약
모든 점·선·행렬이 "스케일 빼고" 정의된다. 계산 결과 마지막에 `w`로 나눠 실제 좌표로 되돌리는 습관.

### 5. RANSAC (이상치 제거)
"무작위 샘플로 모델 추정 → inlier 세기 → 반복 → 최선 채택." 매칭에 outlier가 섞이는 모든 곳에서 사용.
> 등장: 캘리브레이션, PnP, 다시점 기하.

# 부록 C. 한 문장 요약 모음

- **필터링:** 이웃과의 내적.
- **엣지:** 밝기의 1차 미분 (Canny = 블러→미분→NMS→hysteresis).
- **코너:** 구조행렬의 두 고유값이 모두 큰 점.
- **SIFT/HOG:** 그래디언트 방향 히스토그램으로 만든 지문.
- **Hough:** 점들의 투표로 도형 찾기.
- **PCA:** 가장 많이 퍼진 방향만 남기는 SVD 차원축소.
- **Optical flow:** 밝기 보존 + 작은 창 최소제곱 (핵심 행렬은 Harris와 동일).
- **동차좌표:** 무한대를 다루려 차원 +1, up-to-scale.
- **Homography:** 평면↔평면 3×3 매핑.
- **카메라:** P = K[R|t], 픽셀 하나 ↔ 3D 광선 하나.
- **캘리브레이션:** 아는 3D-2D 대응으로 K·왜곡 역산.
- **PnP:** K를 알 때 자세 [R|t] 추정.
- **Fundamental matrix:** 대응점은 epipolar line 위에 있다.
- **삼각측량:** 두 광선의 교차로 3D 복원.
- **SfM/BA:** 많은 사진으로 카메라+3D점 복원 후 재투영 오차 동시 최소화.
- **스테레오:** disparity로 깊이 복원, 에너지 최소화로 dense map.
